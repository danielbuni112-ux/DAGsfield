# Integration Plan: CutAI Features into Open Higgsfield Storyboarding App

## Overview
Integrate CutAI's AI storyboarding capabilities into the existing Open Higgsfield storyboarding app while:
- Keeping manual storyboarding as the primary/default option
- Making AI storyboarding secondary ("under" manual)
- Using the existing cinematic color scheme (amber/gold `#d4a054`)

## 2. Integration Strategy

### Decision: Hybrid Integration
Rather than fully merging or keeping separate, integrate CutAI as a **toggleable mode** within the existing storyboarding page, with AI features appearing below manual features.

### Approach
1. **Extend existing storyboard page** (`storyboard-page`) to have two modes:
   - **Manual Mode** (default, primary): Existing shot-prompt-node and shot-board-node functionality
   - **AI Mode** (secondary): CutAI's script-to-storyboard pipeline

2. **Add new AI Storyboard node** to CineGen node palette for workflow integration

3. **Shared components** where possible to maintain consistent UI

## 3. Implementation Phases

### Phase 1: Backend Integration (CutAI API)
**Goal**: Make CutAI's FastAPI backend available as a service

- [ ] **Add CutAI backend as a module** in `/modules/CutAI-backend/`
- [ ] **Create API wrapper** in `/modules/CineGen/src/lib/cutai-api.ts`
- [ ] **Support dual LLM**: Groq (cloud) and Ollama (local) - reuse existing LLM infrastructure
- [ ] **Database schema**: Create shared SQLite or use existing project database

**API Endpoints needed**:
- `POST /projects` - Create project
- `POST /scripts/generate` - AI script generation from genre/premise
- `POST /scenes/generate` - Scene analysis with shot breakdown
- `GET /projects/:id` - Get project with scenes/shots
- `PATCH /scenes/:id` - Update scene
- `POST /export/pdf` - PDF export

### Phase 2: UI Components
**Goal**: Create AI storyboarding UI components with cinematic theme

- [ ] **Create `AIStoryboardPanel.tsx`** - Main AI storyboarding interface
  - Script generator input (genre + premise)
  - OR raw script textarea
  - Generated scene cards with shot breakdowns

- [ ] **Create `MoodGraph.tsx`** - 4-dimensional mood arc (tension, emotion, energy, darkness) using Recharts

- [ ] **Create `SoundtrackVibes.tsx`** - Per-scene soundtrack suggestions

- [ ] **Create `ShotAnalysis.tsx`** - Camera angles, movements, shot types

- [ ] **Enhance `StoryboardCanvas.tsx`** - Add drag-and-drop reordering (from CutAI)

### Phase 3: Page Integration
**Goal**: Integrate AI mode into existing storyboard page

- [ ] **Modify `storyboard-page.tsx`**:
  - Add tab/mode toggle: "Manual" | "AI Storyboarder"
  - Manual mode: existing functionality (default)
  - AI mode: CutAI-powered workflow

- [ ] **Ensure consistent theming**:
  - Use `--accent: #d4a054` for buttons, highlights
  - Dark backgrounds (`--bg-void: #08090c`, `--bg-base: #0d0f14`)
  - Amber port colors

### Phase 4: Node Integration (Optional/Conditional)
**Goal**: Add AI storyboarder to CineGen node palette

- [ ] **Create `AIStoryboarderNode.tsx`** in `/modules/CineGen/src/components/create/nodes/`
- [ ] **Register in node-registry.ts** as `aiStoryboarder`
- [ ] **Export from nodes/index.ts**

### Phase 5: Feature Enhancements
**Goal**: Additional features from CutAI

- [ ] **Visual Timeline** - React Flow integration for scene flow
- [ ] **PDF Export** - Professional storyboard export
- [ ] **JSON Export** - Data export for external use

## 7. Key Technical Decisions

### Q: Backend deployment?
**A**: Run CutAI FastAPI alongside existing backend. Can be:
- Embedded in existing Express/FastAPI server
- Separate service with CORS
- Serverless functions (Vercel/Railway)

### Q: Database?
**A**: SQLite (CutAI default) is simple but:
- Consider integrating with existing Supabase if available
- Use project-scoped SQLite files

### Q: LLM Provider?
**A**: Support both:
- **Groq** (default, cloud) - fast, no GPU needed
- **Ollama** (local) - privacy, no API costs
- Reuse existing `llm_client.py` from CutAI

### Q: React Flow vs Timeline?
**A**: 
- Existing app uses custom timeline
- CutAI uses React Flow for visual timeline
- **Decision**: Keep existing timeline, optionally add React Flow as visualization mode
