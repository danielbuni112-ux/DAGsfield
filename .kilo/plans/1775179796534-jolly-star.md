# Integration Plan: Adding Remix Go to Higgsfield

This plan details the exact steps and file modifications required to integrate the Remix Go video editor into the existing Open-Higgsfield-AI workspace. This takes into account the current vanilla JS/Vite architecture rather than a Next.js/Express architecture.

## Step 1: Repository Integration

We will add Remix Go as a Git Submodule.
1. Navigate to the root directory `/workspaces/Open-Higgsfield-AI`.
2. Run the submodule commands to add Remix Go into the `apps/` directory:
   ```bash
   git submodule add https://github.com/deangilmoreremix/remix-go apps/remix-go
   git submodule update --init --recursive
   ```

## Step 2: Environment Configuration

Ensure the root `.env` has the required Supabase credentials and environment details:

## Step 3: Server & Proxy Configuration (Vite)

Since Higgsfield uses Vite rather than a custom `server.js` (Next.js/Express), the proxying will be handled directly in `vite.config.js`.

**1. `vite.config.js` Modifications**
Add the proxy routing to Vite's server configuration to proxy `/apps/remix-go` requests to the Remix Go dev server running on port `5173`:
```javascript
// In vite.config.js under `server.proxy`:
'/apps/remix-go': {
    target: 'http://localhost:5173',
    changeOrigin: true,
    rewrite: (path) => path.replace(/^\/apps\/remix-go/, '')
}
```

## Step 4: Package Configuration & Scripts

Update the root `package.json` to handle the monorepo setup and concurrently run both Vite servers.

**1. `package.json` Modifications**
Add the new dependencies and scripts:
```json
{
  "scripts": {
    "dev:all": "concurrently \"npm run dev\" \"npm run dev:remix-go\"",
    "dev:remix-go": "cd apps/remix-go && npm run dev",
    "build:all": "npm run build && npm run build:remix-go",
    "build:remix-go": "cd apps/remix-go && npm run build",
    "test:all": "npm run test && npm run test:remix-go"
  },
  "dependencies": {
    "concurrently": "^7.6.0",
    "wait-on": "^7.0.1"
  }
}
```
*(Note: `http-proxy-middleware` is omitted because Vite provides built-in proxying).*

## Step 5: Infrastructure Additions

Add the necessary multi-app orchestration and reverse proxy files requested.

**1. `docker-compose.yml`**
Create a new Docker compose file to orchestrate the main app, remix-go, and nginx.

**2. `nginx.conf`**
Create a new reverse proxy configuration to route production traffic to `/apps/remix-go/`.

## Step 6: UI and Navigation Integration

We will modify the existing `src/components/Header.js` (the vanilla JS equivalent of the React `Menu.js` requested).

**1. `src/components/Header.js` Modifications**
- Add "Video Editor (Remix Go)" to the `videoDropdownItems` array:
  ```javascript
  const videoDropdownItems = [
    // ... existing items
    { label: 'Video Editor (Remix Go)', route: '/apps/remix-go/' },
  ];
  ```
- Add custom cross-app navigation handling inside the `navigate` routing logic to use `window.location.href` for absolute external app paths (`/apps/remix-go/`) instead of the internal SPA router.
- Implement the requested `sendThemeToRemixGo` functionality using `localStorage` or `sessionStorage` since `window.postMessage` might require a loaded iframe.

The user will need to run the following SQL migration in their Supabase SQL Editor:
