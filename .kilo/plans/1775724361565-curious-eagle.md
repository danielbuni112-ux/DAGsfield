## Objective
Thoroughly integrate 100% of MuAPI's available video and image capabilities across all repositories that require video/image processing, based on comprehensive review of official documentation at https://muapi.ai/docs. ## Prerequisites
1. **Account Setup**: Register at https://muapi.ai/signup and verify email.
2. **API Key**: Generate key at https://muapi.ai/access-keys; store securely in environment variables (e.g., `MUAPI_API_KEY`).
3. **Credit Balance**: Ensure positive balance (>0) for all operations.
4. **Dependencies**: Standard HTTP clients (e.g., axios for JS, requests for Python). For file uploads, support multipart/form-data.
5. **Environment**: Node.js/Python runtime; HTTPS required for all requests.

## Core Setup Steps
1. **Configure Environment**:
   - Set base URL: `https://api.muapi.ai/api/v1`.
   - Add headers: `x-api-key: ${MUAPI_API_KEY}`, `Content-Type: application/json`.
   - For uploads: Use `https://api.muapi.ai/upload_file`.
   - Implement retry logic (exponential backoff, max 3 attempts) for polling.

2. **Authentication Module**:
   - Create utility to append API key to headers.
   - Add balance check before requests (query endpoint if available; assume via trial).
   - Handle 401/402 errors with user notifications.

3. **File Upload Handler**:
   - Implement upload function: POST multipart with `file` key.
   - Validate file types/sizes: Images ≤10MB (jpg/png/webp), Videos ≤50MB (mp4/mov).
   - Parse response for `url`; handle expiry (1 hour).
   - Error handling: 413 for oversized, insufficient balance.

4. **Async Task Manager**:
   - Implement polling for async endpoints: POST request, then GET `/predictions/{id}/result` until `status: "completed"`.
   - Optional webhook integration: Accept POST to custom endpoint with task payload.
   - Handle `failed` status, NSFW flags, timings.

## Feature Implementation Steps

### 1. Image Generation (Flux Dev)
- Endpoint: `POST /flux-dev-image`.
- Inputs: `prompt` (required), optional `image`, `mask_image`, `strength`, etc.
- Implementation: Build request builder, handle async response, display images.
- Validation: Check NSFW array; guard against invalid prompts.

### 2. Video Effects (AI Video Effects, Motion Controls, VFX)
- Endpoint: `POST /generate_wan_ai_effects`.
- Inputs: `prompt`, `image_url`, `name` (preset selector), optional aspect/resolution.
- Implementation: Map UI presets to `name` values; upload images first.
- Validation: Validate uploaded URLs; handle video output.

### 3. Specialized Apps (Face Swap, Upscaling, etc.)
- Endpoints: Various (e.g., `/ai-image-face-swap`).
- Inputs: Vary (e.g., source/target images for swaps).
- Implementation: Create component for each app; chain with uploads.
- Validation: Ensure input quality; parse outputs as URLs.

### 4. Storyboarding
- Endpoint: `POST /api/storyboard/projects`.
- Inputs: Characters (traits/images), scenes/shots.
- Implementation: Integrate into StoryboardPage.js; handle async project creation.
- Validation: Verify character consistency; use webhooks for large projects.

### 5. Audio Integration (Music & Speech)
- Endpoints: Suno, Lipsync, MMAudio.
- Inputs: Prompts for music; URLs for sync.
- Implementation: Add audio generation/sync to video workflows.
- Validation: Match lengths; handle audio formats.

### 6. Workflows and Agents
- Endpoints: `POST /workflow/{id}/run`.
- Inputs: JSON payloads.
- Implementation: CLI integration or UI for workflow building.
- Validation: Test node failures.

### 7. External APIs
- Integration: Via workflows (e.g., Straico, WaveSpeed).
- Implementation: Add nodes for external models.
- Validation: Dynamic schemas; live testing.

## Testing and Validation
1. **Unit Tests**: Mock API responses; test request building, response parsing.
2. **Integration Tests**: End-to-end with real API (use test credits).
   - Upload file → verify URL.
   - Generate image/video → poll/completion.
   - Apply effects → check outputs.
   - Storyboard → verify consistency.
   - Webhook → simulate delivery.
3. **Edge Cases**: Invalid inputs, rate limits (handle 429), NSFW, failures.
4. **Repeatable Tests**: Automate with scripts; confirm all features work.

## Potential Issues and Pitfalls
- Async handling required; no instant results.
- Credit costs undocumented; monitor usage.
- File expiry; re-upload as needed.
- Model differences; test each preset.
- Webhook security; implement signature verification if possible.
