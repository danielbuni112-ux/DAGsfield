# Plan: Integrate Remix Go into Open-Higgsfield-AI

## Overview
This plan outlines the integration of Remix Go (a video editor app) into the existing Open-Higgsfield-AI codebase. The current project uses Vite for building and has an `apps/` directory structure with sub-applications (director, vimax). We'll adapt the provided guide to fit this Vite-based monorepo setup.

## Key Assumptions and Adaptations
- Current setup uses Vite (not Next.js/Express as in the guide)
- No existing server.js or components/Menu.js files
- Integration will involve adding Remix Go as a sub-app in `apps/remix-go/`
- Routing and proxying will need Vite-specific solutions
- Database and environment setup will follow existing patterns

## Step 1: Repository Integration
- **Option A (Recommended):** Add as Git submodule
  - Run: `git submodule add https://github.com/deangilmoreremix/remix-go apps/remix-go`
  - Run: `git submodule update --init --recursive`
- **Option B:** Direct copy
  - Copy Remix Go app to `apps/remix-go/`

## Step 2: File Additions and Modifications
### New Files to Add:
- `apps/remix-go/` (entire Remix Go codebase)
- `.env` (root, with Supabase credentials)
- `docker-compose.yml` (if not existing)
- `nginx.conf` (reverse proxy config)
- `package.monorepo.json` (monorepo config, if needed)

### Files to Modify:
- `package.json` (root): Add monorepo scripts and dependencies
  - Add scripts: `"dev:remix-go": "cd apps/remix-go && npm run dev"`
  - Add dependencies: `concurrently`, `wait-on`, `http-proxy-middleware` (though proxy may need adaptation for Vite)
- Navigation components: Add Remix Go menu item
  - Locate navigation components (likely in `packages/navigation/` or similar)
  - Add dropdown item linking to `/apps/remix-go/`

## Step 3: Vite Configuration for Integration
- Modify `vite.config.js` (root or apps/remix-go) to handle routing/proxying
- Since no Express server, use Vite's proxy or adjust for SPA routing
- Ensure Remix Go can be accessed at `/apps/remix-go/` path

## Step 4: Database Setup
- Run SQL migrations on Supabase
- Use the provided SQL schema for shared tables (projects, media_assets, etc.)
- Ensure compatibility with existing database schema in Higgsfield

## Step 5: Dependency Installation
- Install root-level dependencies: `npm install concurrently wait-on http-proxy-middleware`
- Install Remix Go dependencies: `cd apps/remix-go && npm install`

## Step 6: Build and Dev Scripts Update
- Update `package.json` scripts for monorepo:
  - `"dev": "concurrently \"npm run dev\" \"npm run dev:remix-go\""`
  - `"build": "npm run build && npm run build:remix-go"`
  - `"build:remix-go": "cd apps/remix-go && npm run build"`

## Step 7: Testing and Verification
- Start development servers: `npm run dev`
- Verify navigation to `/apps/remix-go/`
- Check theming consistency and data sharing
- Run tests: `npm run test` (include Remix Go tests)

## Step 8: Production Deployment
- Build all apps: `npm run build:all`
- Deploy to hosting platform (Vercel, Netlify, etc.)
- Set production environment variables
- Configure reverse proxy if needed

## Risks and Considerations
- Routing conflicts in Vite SPA setup
- Dependency conflicts between apps
- Database schema integration
- Theming synchronization across apps
- Performance impact of additional app

## Verification Checklist
- Navigation works
- Apps load correctly
- Data sharing functions
- Video editing features operational
- No console errors
- Performance acceptable
